占位符 对应配置项 说明 示例值
${PORT} port Redis实例端口号，根据节点顺序自动分配 6379, 6380...
${NODE_IP} bind 服务器IP地址，自动填充为服务器组配置的IP 192.168.1.10
${MAX_MEMORY} maxmemory 最大内存限制(MB)，创建集群时设置 4096
${MAXMEMORY_POLICY} maxmemory-policy 缓存淘汰策略，创建集群时选择 allkeys-lru
${REQUIREPASS} requirepass 客户端连接密码，留空则不设置密码 mypassword
${MASTERAUTH} masterauth 从节点连接主节点的密码，通常与requirepass相同 mypassword

Redis配置项说明

以下配置项适用于 Redis 5.x 和 Redis 6.x 所有版本
配置项说明
portRedis服务监听的端口号，默认6379
bind绑定的IP地址，建议设置为服务器内网IP
daemonize是否以守护进程方式运行，建议yes
dir数据文件存储目录
pidfile进程ID文件路径
logfile日志文件路径
maxmemory最大内存限制，达到上限后会根据策略淘汰数据
maxmemory-policy内存淘汰策略：noeviction, allkeys-lru, volatile-lru, allkeys-random, volatile-random, volatile-ttl, allkeys-lfu, volatile-lfu
appendonly是否开启AOF持久化，yes/no
appendfilenameAOF文件名
appendfsync同步策略：always(每次写入), everysec(每秒), no(由OS决定)
saveRDB保存策略，格式：save <秒数> <修改键数>
dbfilenameRDB文件名，默认dump.rdb
cluster-enabled是否开启集群模式，yes/no
cluster-config-file集群配置文件路径
cluster-node-timeout节点超时时间（毫秒）
requirepass客户端连接密码
masterauth从节点连接主节点的密码

以下配置项仅在 Redis 6.x 及以上版本可用
配置项说明
protected-mode保护模式（Redis 6中默认更严格），建议生产环境开启
io-threadsIO线程数，用于多线程IO处理（默认关闭，需显式配置）
io-threads-do-reads是否开启IO多线程读，yes/no
cluster-require-full-coverage是否需要完整槽位覆盖，建议no
cluster-allow-reads-when-down集群宕机时是否允许读操作（Redis 6新增）
aclfileACL配置文件路径，用于精细权限控制（Redis 6新增）
tls-portTLS加密连接端口（Redis 6新增）
tls-cert-fileTLS证书文件路径
tls-key-fileTLS私钥文件路径

占位符说明
${PORT}Redis实例端口号
${NODE_IP}服务器IP地址
${MAX_MEMORY}最大内存限制(MB)，创建集群时设置
${MAXMEMORY_POLICY}缓存淘汰策略，创建集群时选择
${REQUIREPASS}客户端连接密码
${MASTERAUTH}从节点连接主节点的密码
