总内存使用说明
  • 统计对象：仅统计 3个主节点 的内存使用
  • 数据来源：Redis INFO 命令的 used_memory 字段
  • 单位：MB（自动转换为GB显示，大于1024MB时）
  • 计算方式：3个主节点的 used_memory 数值相加（字节转MB）
  • 说明：不包含从节点的内存使用
总内存限制说明
  • 统计对象：仅统计 3个主节点 的内存限制
  • 数据来源：Redis INFO 命令的 maxmemory 字段
  • 配置位置：redis.conf 中的 maxmemory 配置项
  • 单位：MB（自动转换为GB显示，大于1024MB时）
  • 计算方式：3个主节点的 maxmemory 数值相加（字节转MB）
  • 说明：这是集群可用于存储数据的理论最大内存上限
内存占比说明
内存占比 = (内存使用 MB / 内存限制 MB) × 100%
  • 统计对象：仅针对 3个主节点 计算
  • 单位：基于MB计算
  • 颜色说明： < 70% 健康 70% - 90% 警告 > 90% 危险
  • 建议：当占比超过 90% 时，建议扩容或清理数据
总连接数说明
  • 统计对象：仅统计 3个主节点 的客户端连接
  • 数据来源：Redis INFO 命令的 connected_clients 字段
  • 采集方式：每30秒自动采集一次
  • 计算方式：3个主节点的 connected_clients 数值相加
  • 说明：不包含集群内部节点间的连接
状态说明
状态 说明
创建中 集群正在创建或初始化中
运行中 集群正常运行
已停止 集群已停止运行
异常 集群存在异常，需要检查
健康度说明
健康度 = (运行中节点数 / 总节点数) × 100%
健康度范围 评估
≥ 80% 健康 - 集群运行良好
50% - 79% 警告 - 部分节点可能存在问题
< 50% 危险 - 大量节点异常，需立即处理
